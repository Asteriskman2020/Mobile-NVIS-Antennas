1. What is WSPR?

WSPR (Weak Signal Propagation Reporter, pronounced “whisper”) is a digital protocol designed by Joe Taylor, K1JT (Nobel laureate in Physics). It encodes a callsign, Maidenhead grid locator, and transmit power into an ultra-narrowband signal (~6 Hz wide) transmitted in 2-minute cycles.

WSPR can decode signals at SNR as low as −28 dB to −31 dB, far below human hearing. Decoded spots are uploaded to wsprnet.org to build a real-time global map of HF propagation.

No ham radio license is required for receive-only WSPR monitoring. Register with an SWL (Short Wave Listener) identifier on WSPRnet and contribute propagation data using just an RTL-SDR V4 dongle and a simple antenna.

4. RTL-SDR V4 Important Notes

The RTL-SDR V4 has a built-in upconverter (not direct sampling like the V3). Do NOT enable “Direct Sampling” mode in SDR# — it will produce no signal. Simply tune to the HF frequency and it works automatically.
FeatureV4 Behavior
HF ReceptionBuilt-in upconverter (SA612 mixer + 28.8 MHz LO). Just tune to any HF frequency.
Direct SamplingDo NOT enable. Leave disabled.
“Offset Tuning” in SDR#Repurposed: controls the bias tee (4.5V output for active antennas). Not for tuning offset.
IF AGCDo NOT select IF AGC in SDR# — causes issues.
Frequency StabilityBuilt-in TCXO. PPM offset typically 0–1. Allow 30 min warm-up.
HF GainAdjustable via RF Gain slider (unlike V3 direct sampling which had no HF gain control).

Step 1: Install Zadig USB Drivers

  1. Download Zadig and plug in the RTL-SDR V4 dongle.
  2. Right-click Zadig → Run as Administrator.
  3. Go to Options → List All Devices and check it.
  4. From the dropdown, select “Bulk-In, Interface (Interface 0)”.
    You may see “RTL2832UHIDIR”, “RTL2832U”, or “Blog V4” — all are valid.
    Verify USB ID shows 0BDA 2838 00.
  5. Ensure WinUSB is selected as the target driver (right side).
  6. Click “Replace Driver” and accept any publisher warnings.
Do NOT select any device other than the RTL-SDR in Zadig’s dropdown, or you will overwrite that device’s driver (mouse, keyboard, audio, etc.). Always verify the USB ID is 0BDA 2838.

Step 2: Install SDR# (SDRSharp)

  1. Install .NET 8.0 x86 Desktop Runtime and Visual C++ Redistributable (x86) from Microsoft.
  2. Download sdrsharp-x86.zip from airspy.com (choose “Software Defined Radio Package”).
  3. Extract to a folder like C:\SDRSharp.
    Do NOT extract into Program Files (becomes read-only).
  4. Run install-rtlsdr.bat from the extracted folder to download the RTL-SDR driver files.
  5. Launch SDRSharp.dotnet8.exe.
  6. Click the hamburger menu (top-left) and set Source to “RTL-SDR USB”.
  7. Press Play ▶ to verify reception. You should see the spectrum waterfall.
  8. Open the Source settings panel and increase RF Gain (default is zero).
If SDR# doesn’t detect the dongle, re-run Zadig. If you moved the dongle to a different USB port, you may need to reinstall the driver for that port.

Step 3: Install VB-Cable (Virtual Audio)

  1. Download from vb-audio.com/Cable.
  2. Extract the ZIP file.
  3. Right-click VBCABLE_Setup_x64.exe → Run as Administrator.
  4. Follow the prompts to install.
  5. Restart Windows (the virtual audio devices won’t appear until after reboot).
  6. Verify: Settings → System → Sound — you should see “CABLE Input” and “CABLE Output”.
CABLE Input = the virtual microphone (SDR# sends audio here).
CABLE Output = the virtual speaker (WSJT-X reads audio from here).
They are internally connected — audio sent to CABLE Input appears on CABLE Output.

Step 4: Install WSJT-X

  1. Download the latest version from wsjt.sourceforge.io.
  2. Run the Windows installer (wsjtx-X.X.X-win64.exe).
  3. Follow the installation prompts (default settings are fine).
  4. Launch WSJT-X — it will open the Settings dialog on first run.

Step 5: Synchronize PC Clock (NTP)

WSPR decoding requires your PC clock to be accurate within ±1–2 seconds of UTC. The built-in Windows time sync only checks once per week, which is not sufficient.

Install one of these NTP synchronization tools:

ToolNotes
Meinberg NTP Recommended. Runs as background service, auto-starts on boot.
BktTimeSync Lightweight, made for digital modes.
JTSync Simple tool designed for WSJT-X users.
NetTime Keeps clock within milliseconds.

After installation, verify accuracy by comparing with time.is.

Meinberg NTP is the best choice — it installs as a Windows service, syncs continuously, and auto-starts on boot. Once installed, you never need to think about time again.

Step 6: Configure SDR# for WSPR

  1. Launch SDR# and select source “RTL-SDR USB”.
  2. Set demodulation mode to USB (Upper Sideband).
  3. Set filter bandwidth to 2500 Hz (2.5 kHz).
  4. Tune to a WSPR dial frequency, e.g., 14.095600 MHz for 20m.
  5. In Audio settings, change output to “MME: CABLE Input (VB-Audio Virtual Cable)”.
  6. Adjust RF Gain until you see signals on the waterfall without overloading (green/yellow, not red).
  7. Press Play ▶ to begin receiving.
WSPR signals appear within a narrow 200 Hz window at +1400 Hz to +1600 Hz from the dial frequency. On the SDR# waterfall, you may see faint narrow traces during 2-minute transmission windows (starting at even minutes: :00, :02, :04, etc.).
Remember for V4: Do NOT enable Direct Sampling or IF AGC. Leave “Offset Tuning” unchecked unless you want to enable the bias tee for an active antenna.

Step 7: Configure WSJT-X for WSPR

General Settings

  1. Open WSJT-X → File → Settings → General tab.
  2. Enter your Callsign (or SWL identifier, e.g., “SWL-YOURNAME”).
  3. Enter your Grid Locator (6-character Maidenhead, find yours at levinecentral.com).

Audio Settings

  1. Go to File → Settings → Audio tab.
  2. Set Input to: CABLE Output (VB-Audio Virtual Cable)
  3. Leave Output as your default speakers or “None”.

Radio Settings (Simple — No CAT)

  1. Go to File → Settings → Radio tab.
  2. Set Rig to: None
  3. You will manually tune SDR# to the correct WSPR frequency.

Start WSPR Mode

  1. In the WSJT-X main window, select Mode → WSPR.
  2. Select the Band from the dropdown (e.g., “20m”).
  3. Set TX Pct to 0 (receive only!).
  4. Check “Upload spots” to report decoded stations to WSPRnet.
  5. Click Start to begin decoding.
WSJT-X decodes WSPR signals at the end of every 2-minute window. Decoded spots appear with callsign, grid, power, SNR, and frequency offset. Spots are automatically uploaded to WSPRnet if “Upload spots” is checked.

Step 8: Register & Report to WSPRnet

  1. Go to wsprnet.org and click “Create a new account”.
  2. Enter your callsign (or SWL identifier) and email address.
  3. Verify your email and set a password.
  4. In WSJT-X, ensure your callsign matches your WSPRnet account.
  5. With “Upload spots” checked, spots are automatically reported every 2-minute cycle.

Viewing Your Spots

  • WSPRnet Map: wsprnet.org → Map → filter by your callsign
  • WSPR Rocks: Modern alternative at wspr.rocks
  • DX Maps: Visual propagation at dxmaps.com

Antenna Recommendations

Random Wire + 9:1 UnUn
Easiest multi-band option. Run 10–20 m of insulated wire outside, as high as possible. Connect via 9:1 UnUn balun to coax. Even a 5 m wire outdoors will decode WSPR worldwide on 20m.
Dipole / Inverted-V
Best for a single band. Total length = 468 / freq(MHz) in feet. A 20m dipole is ~33 ft total. Hang as inverted-V from a single high point.
Mini Whip / Active Antenna
Compact for limited space. Mount outside away from noise. The V4’s bias tee (4.5V) can power it directly via the “Offset Tuning” checkbox in SDR#.
Magnetic Loop
Best noise rejection for urban environments. Compact, can work indoors. Narrowband (needs tuning per band). Excellent near electronics.
For WSPR receive-only, almost any wire antenna will work. The key is getting it outside and away from electronic noise sources. A simple wire thrown over a tree branch will outperform an indoor antenna every time.

Advanced: CAT Control & Band Hopping

For automatic multi-band monitoring, set up CAT (Computer Aided Transceiver) control so WSJT-X can retune SDR# automatically.

Requirements

  • com0com — Virtual serial port pair (Download)
  • SDR# Calico CAT plugin — Enables CAT control in SDR#

Setup

  1. Install com0com and create a virtual port pair (e.g., COM11 ↔ COM12).
  2. Install the Calico CAT plugin in SDR# (copy DLL, edit Plugins.xml).
  3. In SDR#, enable Calico CAT and assign it to COM11.
  4. In WSJT-X → Settings → Radio, set Rig to “Kenwood TS-2000”, port COM12, baud 19200.
  5. Click “Test CAT” to verify the green indicator lights up.
  6. Enable Band Hopping in WSJT-X → Settings → Frequencies.
Alternative: SDR Console V3 has built-in CAT serial port support, eliminating the need for the Calico plugin. It’s a simpler setup if you prefer SDR Console over SDR#.
