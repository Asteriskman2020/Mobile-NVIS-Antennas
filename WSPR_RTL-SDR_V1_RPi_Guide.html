1. What is WSPR?

WSPR (Weak Signal Propagation Reporter, pronounced “whisper”) is a digital protocol designed by Joe Taylor, K1JT (Nobel laureate in Physics). It encodes a callsign, Maidenhead grid locator, and transmit power into an ultra-narrowband signal (~6 Hz wide) transmitted in 2-minute cycles.

WSPR can decode signals at SNR as low as −28 dB to −31 dB, far below human hearing. Decoded spots are uploaded to wsprnet.org to build a real-time global map of HF propagation.

No ham radio license is required for receive-only WSPR monitoring. Register with an SWL (Short Wave Listener) identifier on WSPRnet and contribute propagation data using just an RTL-SDR dongle, an upconverter, and a simple wire antenna.

How WSPR Works

  • Transmitters send a 110.6-second message every 2 minutes (starting on even minutes)
  • Message contains: callsign, 4-digit grid, dBm power
  • 4-FSK modulation with 1.46 Hz tone spacing
  • Total bandwidth: ~6 Hz — fits inside the 200 Hz WSPR sub-band
  • Receiver decodes and uploads “spots” to wsprnet.org with SNR, frequency offset, and drift

2. RTL-SDR V1 vs V4 — Key Differences

The original RTL-SDR (V1/V2/V3) and the newer V4 have critical differences for HF WSPR reception:

RTL-SDR V1 / V2 / V3

  • Tuner: R820T / R820T2
  • Range: 24 MHz – 1.766 GHz
  • No native HF reception below 24 MHz
  • Requires upconverter for HF bands
  • Upconverter adds 100 or 125 MHz offset
  • Plastic or basic metal case
  • ~$10–25 USD
vs

RTL-SDR Blog V4

  • Tuner: R828D
  • Range: 500 kHz – 1.766 GHz
  • Direct HF sampling via software
  • No upconverter needed
  • No frequency offset required
  • Metal case, improved shielding
  • ~$35 USD
The RTL-SDR V1/V2/V3 cannot tune below 24 MHz natively. All HF WSPR bands (160m through 10m: 1.8–28 MHz) are below this limit. You must use an HF upconverter to shift the signals into the dongle’s tunable range. This guide covers the complete upconverter setup.

Step 1: Flash Raspberry Pi OS

Download Raspberry Pi Imager

  1. Download Raspberry Pi Imager on your PC/Mac
  2. Insert MicroSD card into your computer
  3. Choose OS: Raspberry Pi OS Lite (64-bit) — no desktop needed for headless WSPR
  4. Click the gear/settings icon to pre-configure:
    • Set hostname: wspr-pi
    • Enable SSH with password authentication
    • Set username: pi and a strong password
    • Configure WiFi (SSID + password) if not using Ethernet
    • Set locale/timezone to your location
  5. Write to SD card, eject, insert into Raspberry Pi, power on

Connect via SSH

# From your PC terminal (wait ~60 seconds after power-on):
$ ssh pi@wspr-pi.local
# Or use the Pi's IP address (check your router):
$ ssh pi@192.168.1.xxx

Update the System

$ sudo apt update && sudo apt upgrade -y
Use Raspberry Pi OS Lite (no desktop) for a lean 24/7 WSPR station. It uses less RAM, CPU, and generates less electrical noise.

Step 2: Install RTL-SDR Drivers

Install rtl-sdr Package

$ sudo apt install -y rtl-sdr librtlsdr-dev

Blacklist DVB-T Kernel Driver (Important!)

Linux auto-loads a DVB-T TV driver that blocks rtl-sdr. You must blacklist it:

$ sudo bash -c 'cat > /etc/modprobe.d/blacklist-rtlsdr.conf <<EOF
blacklist dvb_usb_rtl28xxu
blacklist rtl2832
blacklist rtl2830
EOF'

$ sudo reboot

Plug in RTL-SDR V1 & Verify

# After reboot, plug RTL-SDR V1 into a USB port
$ rtl_test -t

# Expected output:
Found 1 device(s):
  0: Realtek, RTL2838UHIDIR, SN: 00000001

Using device 0: Generic RTL2832U OEM
Found Rafael Micro R820T tuner
If rtl_test says “No devices found” — check the blacklist file is saved correctly, reboot again, and try a different USB port. On Pi 4/5, try the USB 2.0 ports (black, not blue).

Check Dongle Serial Number

If you have multiple RTL-SDR dongles, set unique serial numbers:

# View current serial:
$ rtl_eeprom

# Set a new serial (optional, for multi-dongle setups):
$ rtl_eeprom -s 00000001

Step 3: Upconverter Setup

This is the critical step that differs from RTL-SDR V4. The upconverter shifts HF signals (1–30 MHz) upward by a fixed offset so the R820T2 tuner can receive them.

Physical Connections

SMA cable SMA male-male HF Antenna ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∂ [ Upconverter ] ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∂ [ RTL-SDR V1 ] ‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∂ USB to Pi HF IN RF OUT SMA IN USB ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê ‚îÇ Ham It Up ‚îÇ ‚îÇ LO: 125 MHz ‚îÇ ‚îÇ Power: USB ‚îÇ ‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
  1. Connect your HF antenna to the upconverter’s HF IN (or RF IN) port via SMA cable
  2. Connect the upconverter’s IF OUT (or RF OUT) port to the RTL-SDR V1 SMA input with an SMA male-to-male cable
  3. Connect RTL-SDR V1 to Raspberry Pi via USB (use a 1–2 m extension cable to reduce noise)
  4. Power the upconverter:
    • Ham It Up v1.3: powered via USB passthrough from RTL-SDR — no extra cable needed
    • Ham It Up Nano: connect separate 5V micro-USB power
    • SpyVerter R2: powered via bias-tee — enable with rtl_biast -b 1
    • Generic: connect separate 5V power supply
  5. Make sure the upconverter’s passband switch (if present) is set to HF mode, not VHF passthrough

Understanding the Frequency Offset

The upconverter’s local oscillator (LO) frequency is added to every signal passing through it:

WSPR Band Actual Frequency With 125 MHz Upconverter With 100 MHz Upconverter
160 m 1.836600 MHz 126.836600 MHz 101.836600 MHz
80 m 3.568600 MHz 128.568600 MHz 103.568600 MHz
40 m 7.038600 MHz 132.038600 MHz 107.038600 MHz
30 m 10.138700 MHz 135.138700 MHz 110.138700 MHz
20 m 14.095600 MHz 139.095600 MHz 114.095600 MHz
17 m 18.104600 MHz 143.104600 MHz 118.104600 MHz
15 m 21.094600 MHz 146.094600 MHz 121.094600 MHz
10 m 28.124600 MHz 153.124600 MHz 128.124600 MHz
You must tell the software about your upconverter’s LO frequency! If the offset is wrong, the receiver will tune to the wrong frequency and decode nothing. Common offsets: 125000000 (Ham It Up), 120000000 (SpyVerter), 100000000 (generic Chinese). Check your upconverter’s documentation for the exact LO frequency.

Step 4: Test HF Reception

Before installing wsprdaemon, verify that the RTL-SDR + upconverter can receive HF signals:

Quick Test with rtl_fm

# Install sox for audio playback (optional, for audio test)
$ sudo apt install -y sox

# Test 20m WSPR band with 125 MHz upconverter
# Actual WSPR: 14.095600 MHz ‚Üí Tune to: 14.095600 + 125.000 = 139.095600 MHz
$ rtl_fm -f 139095600 -M usb -s 12000 -g 40 - | \
    sox -t raw -r 12000 -e signed -b 16 -c 1 - -n stats

# You should see signal level statistics updating.
# If "RMS Pk" shows values above -40 dB, your antenna is picking up signals.
# Press Ctrl+C to stop.

Test with rtl_power (Spectrum Scan)

# Scan 20m band region (with 125 MHz offset) for 2 minutes
$ timeout 120 rtl_power -f 139000000:139200000:100 -g 40 -i 10 /tmp/20m_scan.csv

# Check if data was captured:
$ wc -l /tmp/20m_scan.csv
# Should show many lines of data

Verify Gain Setting

# List available gain values for your RTL-SDR:
$ rtl_test -t 2>&1 | grep "Supported gain"

# Typical R820T2 gains (dB):
Supported gain values: 0.0, 0.9, 1.4, 2.7, 3.7, 7.7, 8.7, 12.5,
  14.4, 15.7, 16.6, 19.7, 20.7, 22.9, 25.4, 28.0, 29.7, 32.8,
  33.8, 36.4, 37.2, 38.6, 40.2, 42.1, 43.4, 43.9, 44.5, 48.0, 49.6

# For WSPR, start with gain 40-44 dB. Adjust if needed.
If you hear/see nothing, double-check: (1) upconverter is powered on, (2) SMA cables are tight, (3) antenna is connected, (4) frequency offset is correct. Try a broadcast AM station first (e.g., tune to 125.0 + your local AM frequency in MHz) to confirm the upconverter works.

Step 5: Install wsprdaemon

wsprdaemon is the recommended all-in-one WSPR monitoring solution. It handles RTL-SDR control, WSPR decoding, and spot uploading automatically.

Install Dependencies

$ sudo apt install -y git build-essential libfftw3-dev \
    libsamplerate0-dev libsndfile1-dev curl bc sox

Clone & Install wsprdaemon

$ cd ~
$ git clone https://github.com/rrobinett/wsprdaemon.git
$ cd wsprdaemon

# Run the installer (compiles wsprd decoder and helper tools)
$ ./wsprdaemon.sh -i

# This may take 5-15 minutes on Pi 3/4. It will:
# - Compile wsprd (WSPR decoder)
# - Compile kiwirecorder (if needed)
# - Set up cron job for auto-start
# - Create default config file
wsprdaemon supports RTL-SDR dongles natively and handles the frequency offset for upconverters. It runs as a background daemon, automatically starts on boot, and uploads spots every 2-minute cycle.

Step 6: Configure wsprdaemon

This is the most important step. Edit the configuration file to match your station:

$ cd ~/wsprdaemon
$ nano wsprdaemon.conf

Key Configuration Settings

### Station Identity ###
MY_CALL="YOUR_CALLSIGN"    # Your callsign or SWL ID (e.g., "HS2GJG")
MY_GRID="OK03xx"            # Your 6-character Maidenhead grid locator

### Receiver Definition ###
# Format: "NAME  TYPE  DEVICE_ID  [CONVERTER_OFFSET]"
# The CONVERTER_OFFSET is your upconverter's LO frequency in Hz!
RECEIVER_LIST=(
  "RTL_V1  RTL-SDR  0  125000000"
)
#                       ‚Üë‚Üë‚Üë‚Üë‚Üë‚Üë‚Üë‚Üë‚Üë
# This is the upconverter offset in Hz:
#   125000000 = 125 MHz (Ham It Up)
#   120000000 = 120 MHz (SpyVerter R2)
#   100000000 = 100 MHz (generic Chinese)

### Band Schedule ###
# Which bands to monitor (wsprdaemon hops between them)
WSPR_SCHEDULE=(
  "RTL_V1  20m"    # 14.095600 MHz ‚Äî daytime, most active
  "RTL_V1  40m"    # 7.038600 MHz  ‚Äî nighttime
  "RTL_V1  30m"    # 10.138700 MHz ‚Äî day & night
)

### Upload Settings ###
WSPRNET_UPLOAD="yes"      # Upload spots to wsprnet.org
The CONVERTER_OFFSET value is critical! This tells wsprdaemon to add 125000000 Hz (or your upconverter’s LO frequency) when tuning the RTL-SDR. Without this, it will tune to the wrong frequency and decode nothing. Verify your upconverter’s exact LO frequency from its documentation.

Example Configurations

Single-Band (20m only, simplest)

MY_CALL="HS2GJG"
MY_GRID="OK03fb"

RECEIVER_LIST=(
  "RTL_V1  RTL-SDR  0  125000000"
)

WSPR_SCHEDULE=(
  "RTL_V1  20m"
)

Multi-Band (time-shared hopping)

MY_CALL="HS2GJG"
MY_GRID="OK03fb"

RECEIVER_LIST=(
  "RTL_V1  RTL-SDR  0  125000000"
)

# wsprdaemon hops between bands each 2-minute cycle
WSPR_SCHEDULE=(
  "RTL_V1  20m"
  "RTL_V1  40m"
  "RTL_V1  30m"
  "RTL_V1  80m"
)

Generic 100 MHz Upconverter

RECEIVER_LIST=(
  "RTL_V1  RTL-SDR  0  100000000"
)

Gain Settings (Optional)

# If default gain is too high/low, add to config:
RTL_SDR_GAIN="42.1"   # dB ‚Äî typical good value for HF

# Or set to "auto" for automatic gain:
RTL_SDR_GAIN="auto"
Start with 20m band only during daytime for your first test. It’s the most active WSPR band and you should see decoded stations within the first 2–4 minute cycle if everything is working.

Step 7: Register on WSPRnet

  1. Go to wsprnet.org
  2. Click “Create Account” (top right corner)
  3. Enter your callsign (or SWL ID like “SWL-YourName”) and email address
  4. Verify your email and set a password
  5. Your decoded spots will appear on the global WSPR map
No ham license required! WSPR listening is receive-only. You can register with any identifier. Licensed hams use their callsign; SWLs use “SWL-Name” or similar.

Find Your Grid Locator

Visit levinecentral.com/ham/grid_square.php or search “Maidenhead grid locator” — enter your address to get your 6-character grid (e.g., OK03fb). Use this in your wsprdaemon config.

Step 8: NTP Time Sync (Critical!)

WSPR decoding requires time accuracy within ±2 seconds. The Raspberry Pi has no real-time clock (RTC) — it relies entirely on NTP (Network Time Protocol) to stay synchronized.

Verify NTP Is Active

$ timedatectl status

# Look for these two lines:
System clock synchronized: yes
            NTP service: active

If NTP Is Not Active

$ sudo timedatectl set-ntp true
$ sudo systemctl enable systemd-timesyncd
$ sudo systemctl start systemd-timesyncd

Set Correct Timezone

$ sudo timedatectl set-timezone Asia/Bangkok
# Or use raspi-config for interactive selection:
$ sudo raspi-config   # ‚Üí Localisation ‚Üí Timezone

Optional: Install chrony for Better Accuracy

# chrony is more accurate than systemd-timesyncd
$ sudo apt install -y chrony
$ sudo systemctl enable chrony
$ sudo systemctl start chrony

# Check sync status:
$ chronyc tracking
# Look for "System time" offset ‚Äî should be < 0.1 seconds
If time is off by more than 2 seconds, WSPR decoding will fail completely. Always verify NTP sync before starting wsprdaemon. If your Pi loses internet, the clock will drift and spots will stop.

Step 9: Antenna Setup

Random Wire + 9:1 UnUn
Easiest option. Run 10–20 m of insulated wire outside, as high as possible. Connect to a 9:1 UnUn balun → short coax → SMA adapter → upconverter HF IN. Add a ground wire or counterpoise for better performance.
$10–25
Dipole / Inverted-V
Best for single band. Total length = 468 / freq(MHz) in feet. Example: 20m dipole = 33 ft total. Hang in inverted-V from a single support. Feed with coax + 1:1 balun.
$15–30
Mini Whip / Active Antenna
Compact & broadband. Great for limited space. Mount outside, away from noise sources. Needs 5V power (some use bias-tee). Wideband coverage 10 kHz–30 MHz.
$15–40
Magnetic Loop
Best noise rejection. Excellent for noisy urban environments. Compact, can work indoors. Narrowband — tune to your target WSPR band.
$30–100+
WSPR is incredibly sensitive — even a 5 m wire strung out a window will decode stations worldwide on 20m during daytime. Start simple, upgrade later. The antenna outdoors matters more than the antenna type.
The RTL-SDR V1 + upconverter is more susceptible to strong signal overload than the V4. If you’re near AM broadcast towers, you may need a high-pass filter (cut-off ~1.8 MHz) between the antenna and upconverter to block MW interference.

Step 10: Start & Monitor

Start wsprdaemon

$ cd ~/wsprdaemon

# Start all configured receivers:
$ ./wsprdaemon.sh -a

# Check status:
$ ./wsprdaemon.sh -s

# View live decoded spots:
$ tail -f /tmp/wsprdaemon/wsprd.log

# Stop all receivers:
$ ./wsprdaemon.sh -z

Verify Spots Are Being Uploaded

  1. Wait 4–6 minutes for at least two complete 2-minute cycles
  2. Go to wsprnet.org/drupal/wsprnet/spots
  3. Search for your callsign in the “Reporter” field
  4. You should see decoded spots with transmitter callsign, SNR, frequency, and distance

Auto-Start on Boot

wsprdaemon automatically adds itself to cron during installation. Verify:

$ crontab -l
# Should show a wsprdaemon entry like:
*/2 * * * * /home/pi/wsprdaemon/wsprdaemon.sh -a >/dev/null 2>&1

Alternative: systemd Service (More Robust)

$ sudo nano /etc/systemd/system/wspr.service
[Unit]
Description=WSPR RTL-SDR V1 Receiver Station
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/wsprdaemon
ExecStart=/home/pi/wsprdaemon/wsprdaemon.sh -a
ExecStop=/home/pi/wsprdaemon/wsprdaemon.sh -z
Restart=on-failure
RestartSec=30

[Install]
WantedBy=multi-user.target
$ sudo systemctl daemon-reload
$ sudo systemctl enable wspr.service
$ sudo systemctl start wspr.service
$ sudo systemctl status wspr.service
After a power outage, your Pi will reboot and automatically resume WSPR monitoring. wsprdaemon keeps its state and picks up where it left off.

Alternative: WSJT-X Method

If you prefer a GUI or want to experiment, you can use WSJT-X with virtual audio routing.

Install WSJT-X + PulseAudio

$ sudo apt install -y wsjtx pulseaudio pulseaudio-utils

# Create virtual audio sink:
$ pactl load-module module-null-sink \
    sink_name=wspr_audio \
    sink_properties=device.description=WSPR_Audio

Pipe RTL-SDR Audio to WSJT-X

# Example: 20m WSPR with 125 MHz upconverter
# Tune frequency = 14095600 + 125000000 = 139095600
$ rtl_fm -f 139095600 -M usb -s 12000 -g 40 - | \
    pacat --playback --device=wspr_audio \
    --rate=12000 --format=s16le --channels=1

Configure WSJT-X

  • Audio Input: Monitor of WSPR_Audio (PulseAudio)
  • Mode: WSPR
  • TX %: 0% (receive only!)
  • Callsign + Grid: your station info
  • Upload spots: checked
The WSJT-X method requires a desktop environment or VNC. Enable VNC: sudo raspi-config → Interface Options → VNC → Enable. wsprdaemon is better for headless 24/7 operation.

Troubleshooting

No Spots Decoded

  • Check NTP sync: timedatectl status — must show “synchronized: yes”
  • Verify RTL-SDR detected: rtl_test -t
  • Check upconverter power: Is the LED on? Is it receiving 5V?
  • Verify frequency offset: Confirm your upconverter’s exact LO frequency matches the config
  • Check wsprdaemon logs: tail -f /tmp/wsprdaemon/*.log
  • Check antenna: Is it connected? Is it outside? Tighten all SMA connectors
  • Try 20m daytime: Most active band, easiest to get first spots
  • Band propagation: Check WSPRnet map to see if others are receiving on your target band

USB / Power Issues

  • Use the official Pi power supply (5V 3A for Pi 4/5)
  • Check for undervoltage: dmesg | grep -i voltage
  • Use a powered USB hub if the upconverter draws power from USB
  • Prefer USB 2.0 ports on Pi 4/5 (black, not blue) — less noise on HF
  • Use a USB extension cable (1–2 m) to separate RTL-SDR + upconverter from Pi

Noisy HF Reception

  • RTL-SDR V1 is more susceptible to noise than V4 (plastic case, less shielding)
  • USB extension cable (1–2 m) is the #1 fix — separates dongle from Pi CPU noise
  • Add ferrite chokes (snap-on) to USB cable, power cable, and coax
  • Wrap RTL-SDR in aluminum foil or put it in a metal enclosure for shielding
  • Keep antenna feed away from Pi, monitor, and power supply
  • If near AM broadcast towers, add a high-pass filter (cutoff ~1.8 MHz) at antenna input
  • Try lower gain: RTL_SDR_GAIN="32.8" in wsprdaemon.conf

High CPU / Temperature

  • Check: htop or vcgencmd measure_temp
  • Use a heatsink or fan case for Pi 4/5
  • Reduce number of bands if CPU > 80%
  • Pi OS Lite uses much less resources than Desktop
  • Pi 3B+ may struggle with 3+ bands — stick to 1–2 bands
