1. What is WSPR?

WSPR (Weak Signal Propagation Reporter, pronounced “whisper”) is a digital protocol designed by Joe Taylor, K1JT (Nobel laureate in Physics). It encodes a callsign, Maidenhead grid locator, and transmit power into an ultra-narrowband signal (~6 Hz wide) transmitted in 2-minute cycles.

WSPR can decode signals at SNR as low as −28 dB to −31 dB, far below human hearing. Decoded spots are uploaded to wsprnet.org to build a real-time global map of HF propagation.

No ham radio license is required for receive-only WSPR monitoring. Register with an SWL (Short Wave Listener) identifier on WSPRnet and contribute propagation data using just an RTL-SDR V4 dongle and a simple wire antenna.

How WSPR Works

  • Transmitters send a 110.6-second message every 2 minutes (starting on even minutes)
  • Message contains: callsign, 4-digit grid, dBm power
  • 4-FSK modulation with 1.46 Hz tone spacing
  • Total bandwidth: ~6 Hz — fits inside the 200 Hz WSPR sub-band
  • Receiver decodes and uploads “spots” to wsprnet.org with SNR, frequency offset, and drift

2. Why RTL-SDR V4 for WSPR?

The RTL-SDR Blog V4 is the ideal budget WSPR receiver because it has native HF support:

RTL-SDR V1 / V2 / V3

  • Tuner: R820T / R820T2
  • Range: 24 MHz – 1.766 GHz
  • Cannot receive HF natively
  • Requires external upconverter (+$30–50)
  • Extra frequency offset configuration
  • Plastic case, poor shielding
  • ~$10–25 USD
vs

RTL-SDR Blog V4

  • Tuner: R828D
  • Range: 500 kHz – 1.766 GHz
  • Direct HF sampling!
  • No upconverter needed
  • No frequency offset ‚Äî tune directly
  • Metal case, improved shielding
  • ~$35 USD
The V4’s direct HF sampling mode and metal case make it the simplest and cleanest WSPR setup. Plug in, install software, tune to the WSPR frequency — done. No upconverter, no offset math.

Step 1: Flash Raspberry Pi OS

Download Raspberry Pi Imager

  1. Download Raspberry Pi Imager on your PC/Mac
  2. Insert MicroSD card into your computer
  3. Choose OS: Raspberry Pi OS Lite (64-bit) — no desktop needed for headless WSPR
  4. Click the gear/settings icon to pre-configure:
    • Set hostname: wspr-pi
    • Enable SSH with password authentication
    • Set username: pi and a strong password
    • Configure WiFi (SSID + password) if not using Ethernet
    • Set locale/timezone to your location
  5. Write to SD card, eject, insert into Raspberry Pi, power on

Connect via SSH

# From your PC terminal (wait ~60 seconds after power-on):
$ ssh pi@wspr-pi.local
# Or use the Pi's IP address (check your router):
$ ssh pi@192.168.1.xxx

Update the System

$ sudo apt update && sudo apt upgrade -y
Use Raspberry Pi OS Lite (no desktop) for a lean 24/7 WSPR station. It uses less RAM, CPU, and generates less electrical noise. Connect via SSH only.

Step 2: Install RTL-SDR Drivers

Install rtl-sdr Package

$ sudo apt install -y rtl-sdr librtlsdr-dev

Blacklist DVB-T Kernel Driver (Important!)

Linux auto-loads a DVB-T TV driver that blocks rtl-sdr. You must blacklist it:

$ sudo bash -c 'cat > /etc/modprobe.d/blacklist-rtlsdr.conf <<EOF
blacklist dvb_usb_rtl28xxu
blacklist rtl2832
blacklist rtl2830
EOF'

$ sudo reboot

Plug in RTL-SDR V4 & Verify

# After reboot, plug RTL-SDR V4 into a USB port
$ rtl_test -t

# Expected output:
Found 1 device(s):
  0: Realtek, RTL2838UHIDIR, SN: 00000001

Using device 0: Generic RTL2832U OEM
No E4000 tuner found, using default gains.
If rtl_test says “No devices found” — check the blacklist file, reboot again, and try a different USB port. On Pi 4/5, try the USB 2.0 ports (black, not blue).

Step 3: Test HF Reception

Before installing wsprdaemon, verify the V4 can receive HF signals directly:

Quick Test with rtl_fm

# Install sox for audio
$ sudo apt install -y sox

# Tune directly to 20m WSPR frequency ‚Äî no offset needed!
$ timeout 10 rtl_fm -f 14095600 -M usb -s 12000 -g 30 - | \
    sox -t raw -r 12000 -e signed -b 16 -c 1 - -n stats

# You should see signal level statistics.
# If "RMS Pk" shows values above -40 dB, your antenna is picking up signals.

Test with rtl_power (Spectrum Scan)

# Scan 20m WSPR region for 2 minutes
$ timeout 120 rtl_power -f 14000000:14200000:100 -g 30 -i 10 /tmp/20m_scan.csv

# Check if data was captured:
$ wc -l /tmp/20m_scan.csv
The V4’s direct HF tuning means you use the actual WSPR frequency — no offset calculation needed. This is the biggest advantage over V1/V2/V3 dongles.

Step 5A: Install wsprdaemon (Recommended)

wsprdaemon is the recommended all-in-one WSPR monitoring solution for headless Raspberry Pi stations.

Install Dependencies

$ sudo apt install -y git build-essential libfftw3-dev \
    libsamplerate0-dev libsndfile1-dev curl bc sox

Clone & Install wsprdaemon

$ cd ~
$ git clone https://github.com/rrobinett/wsprdaemon.git
$ cd wsprdaemon

# Run the installer (compiles wsprd decoder)
$ ./wsprdaemon.sh -i

# Takes 5-15 minutes on Pi 3/4. It will:
# - Compile wsprd (WSPR decoder)
# - Set up cron job for auto-start
# - Create default config file

Configure wsprdaemon.conf

$ nano wsprdaemon.conf
### Station Identity ###
MY_CALL="YOUR_CALLSIGN"    # Your callsign or SWL ID
MY_GRID="OK03xx"            # Your 6-character Maidenhead grid

### Receiver Definition ###
# RTL-SDR V4 ‚Äî NO converter offset needed!
RECEIVER_LIST=(
  "RTL_V4  RTL-SDR  0"
)

### Band Schedule ###
WSPR_SCHEDULE=(
  "RTL_V4  20m"    # 14.095600 MHz ‚Äî daytime, most active
  "RTL_V4  40m"    # 7.038600 MHz  ‚Äî nighttime
  "RTL_V4  30m"    # 10.138700 MHz ‚Äî day & night
)

### Upload Settings ###
WSPRNET_UPLOAD="yes"
Notice there is no converter offset in the RECEIVER_LIST ‚Äî the V4 tunes directly to HF. Compare with V1 setup which requires "RTL_V1 RTL-SDR 0 125000000".

Single-Band Example (20m only)

MY_CALL="HS2GJG"
MY_GRID="OK03fb"

RECEIVER_LIST=(
  "RTL_V4  RTL-SDR  0"
)

WSPR_SCHEDULE=(
  "RTL_V4  20m"
)

Multi-Band Example (time-shared hopping)

MY_CALL="HS2GJG"
MY_GRID="OK03fb"

RECEIVER_LIST=(
  "RTL_V4  RTL-SDR  0"
)

WSPR_SCHEDULE=(
  "RTL_V4  20m"
  "RTL_V4  40m"
  "RTL_V4  30m"
  "RTL_V4  80m"
)
Start with 20m band only during daytime for your first test. It’s the most active WSPR band and you should see decoded stations within the first 2–4 minute cycle.

Step 5B: Alternative ‚Äî WSJT-X Method

If you prefer a GUI or want to experiment:

Install WSJT-X + PulseAudio

$ sudo apt install -y wsjtx pulseaudio pulseaudio-utils

# Create virtual audio sink:
$ pactl load-module module-null-sink \
    sink_name=wspr_audio \
    sink_properties=device.description=WSPR_Audio

Pipe RTL-SDR V4 Audio to WSJT-X

# V4 tunes directly ‚Äî no offset needed!
$ rtl_fm -f 14095600 -M usb -s 12000 -g 30 - | \
    pacat --playback --device=wspr_audio \
    --rate=12000 --format=s16le --channels=1

Configure WSJT-X

  • Audio Input: Monitor of WSPR_Audio (PulseAudio)
  • Mode: WSPR
  • TX %: 0% (receive only!)
  • Callsign + Grid: your station info
  • Upload spots: checked
The WSJT-X method requires a desktop environment or VNC. Enable VNC: sudo raspi-config → Interface Options → VNC → Enable. wsprdaemon is better for headless 24/7 operation.

Step 6: Register on WSPRnet

  1. Go to wsprnet.org
  2. Click “Create Account” (top right corner)
  3. Enter your callsign (or SWL ID) and email address
  4. Verify your email and set a password
  5. Your decoded spots will appear on the global WSPR map
No ham license required! WSPR listening is receive-only. You can register with any identifier. Licensed hams use their callsign; SWLs use “SWL-Name” or similar.

Find Your Grid Locator

Visit levinecentral.com/ham/grid_square.php — enter your address to get your 6-character Maidenhead grid (e.g., OK03fb).

Step 7: NTP Time Sync (Critical!)

WSPR decoding requires time accuracy within ±2 seconds. The Raspberry Pi has no real-time clock — it relies entirely on NTP.

Verify NTP Is Active

$ timedatectl status

# Look for:
System clock synchronized: yes
            NTP service: active

If NTP Is Not Active

$ sudo timedatectl set-ntp true
$ sudo systemctl enable systemd-timesyncd
$ sudo systemctl start systemd-timesyncd

Set Correct Timezone

$ sudo timedatectl set-timezone Asia/Bangkok
# Or use raspi-config:
$ sudo raspi-config   # ‚Üí Localisation ‚Üí Timezone

Optional: Install chrony for Better Accuracy

$ sudo apt install -y chrony
$ sudo systemctl enable chrony
$ sudo systemctl start chrony

# Check sync:
$ chronyc tracking
If time is off by more than 2 seconds, WSPR decoding will fail completely. Always verify NTP sync before starting wsprdaemon.

Step 8: Antenna Setup

Random Wire + 9:1 UnUn
Easiest option. 10–20 m insulated wire outside, as high as possible. Connect to 9:1 UnUn balun → coax → SMA → RTL-SDR V4. Add ground wire or counterpoise.
$10–25
Dipole / Inverted-V
Best for single band. Length = 468 / freq(MHz) feet. 20m dipole = 33 ft total. Feed with coax + 1:1 balun.
$15–30
Mini Whip / Active Antenna
Compact & broadband. Mount outside, away from noise. V4 has software bias-tee to power it: rtl_biast -b 1
$15–40
Magnetic Loop
Best noise rejection. Excellent for noisy urban environments. Compact, works indoors. Tune to your target WSPR band.
$30–100+
WSPR is incredibly sensitive ‚Äî even a 5 m wire out a window will decode worldwide stations on 20m during daytime. Start simple, upgrade later.

Step 9: Start & Monitor

Start wsprdaemon

$ cd ~/wsprdaemon

# Start all configured receivers:
$ ./wsprdaemon.sh -a

# Check status:
$ ./wsprdaemon.sh -s

# View live decoded spots:
$ tail -f /tmp/wsprdaemon/wsprd.log

# Stop all receivers:
$ ./wsprdaemon.sh -z

  1. Wait 4–6 minutes for at least two complete 2-minute cycles
  2. Go to wsprnet.org/drupal/wsprnet/spots
  3. Search for your callsign in the “Reporter” field
  4. You should see decoded spots with transmitter callsign, SNR, frequency, and distance

Auto-Start on Boot

wsprdaemon automatically adds itself to cron during installation. Verify:

$ crontab -l
# Should show a wsprdaemon entry

Alternative: systemd Service (More Robust)

$ sudo nano /etc/systemd/system/wspr.service
[Unit]
Description=WSPR RTL-SDR V4 Receiver Station
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/wsprdaemon
ExecStart=/home/pi/wsprdaemon/wsprdaemon.sh -a
ExecStop=/home/pi/wsprdaemon/wsprdaemon.sh -z
Restart=on-failure
RestartSec=30

[Install]
WantedBy=multi-user.target
$ sudo systemctl daemon-reload
$ sudo systemctl enable wspr.service
$ sudo systemctl start wspr.service
$ sudo systemctl status wspr.service
After a power outage, your Pi will reboot and automatically resume WSPR monitoring.

Troubleshooting

No Spots Decoded

  • Check NTP sync: timedatectl status ‚Äî must show “synchronized: yes”
  • Verify RTL-SDR detected: rtl_test -t
  • Check wsprdaemon logs: tail -f /tmp/wsprdaemon/*.log
  • Verify antenna is connected and placed outside
  • Try 20m band during daytime first (most active)
  • Check band propagation: WSPRnet map

USB / Power Issues

  • Use the official Pi power supply (5V 3A for Pi 4/5)
  • Check for undervoltage: dmesg | grep -i voltage
  • Use a powered USB hub if using multiple RTL-SDRs
  • On Pi 4/5, prefer USB 2.0 ports (black) ‚Äî less noise on HF
  • Use a USB extension cable (1–2 m) to separate dongle from Pi

Noisy HF Reception

  • USB extension cable (1–2 m) is the #1 fix
  • Add ferrite chokes on USB cable and power cable
  • Keep antenna feed away from Pi and power supply
  • V4’s metal case helps, but still benefits from distance to Pi
  • Lower the gain if AGC is causing noise: RTL_SDR_GAIN="30"

High CPU / Temperature

  • Check: htop or vcgencmd measure_temp
  • Use a heatsink or fan case for Pi 4/5
  • Reduce number of bands if CPU > 80%
  • Pi OS Lite uses much less resources than Desktop
